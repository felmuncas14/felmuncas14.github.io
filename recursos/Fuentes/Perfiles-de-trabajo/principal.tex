    \section{Gestión de perfiles}
    En mi carpeta \texttt{$\sim$/Documentos/Gestión\ del\ sistema/Perfiles} se
    encuentran unos guiones en Bash que inician automáticamente el perfiles
    deseado. Cada guion tiene un nombre descriptivo de lo que almacena.
    Debe estar instalado el paquete \texttt{podman-toolbox}, y se puede
    comprobar que ha funcionado porque el nombre de la máquina a partir de
    ese momento es toolbox.

    \section{Instalación}
    Se instaló desde el gestor de paquetes con
    \texttt{sudo apt install podman-toolbox}.

    \section{Creación de un nuevo perfil}
    Para crear un nuevo perfil primero tengo que ejecutar el comando:

    \texttt{toolbox create --distro ubuntu/arch/fedora --release 22.04/rolling/43f}

    Y para meterme en el contenedor recién creado hago:

    \texttt{toolbox enter ubuntu/arch/fedora-toolbox-22.04/rolling/43f}

    Lo \textbf{más importante} es modificar antes que nada la variable \texttt{HOME},
    con el comando \texttt{export HOME="/mi/ruta/HOME/del/perfil"}. En esa ruta
    es donde se guardan los archivos de configuración por defecto de las aplicaciones.
    Y puede interferir con mi configuración si dejo la que hay por defecto (pues
    usaría la misma carpeta \texttt{.config} que mi ordenador).

    La única forma de hacer esto ha sido añadir el siguiente código en
    mi \texttt{.bashrc}, gracias al usuario inikishev por compartirlo en
    \href{https://github.com/containers/toolbox/issues/348}{este enlace},
    que está archivado \href{https://web.archive.org/web/20260317144147/https://github.com/containers/toolbox/issues/348}
    {aquí}:

    \begin{minted}{bash}
# Check if we're running inside a toolbox
TOOLBOX_NAME=""
if [ -f "/run/.containerenv" ]; then
	TOOLBOX_NAME=$(sed -nr 's/^name="(.*)"$/\1/p' /run/.containerenv)
        HOME="$HOME/.toolbox/homedir/$TOOLBOX_NAME/$USER"
        mkdir -p "$HOME"
        bash
        exit 0
fi
    \end{minted}

    Lo ideal es ahora crear en mi ordenador una variable para referenciar el
    directorio anterior del contenedor. Para poder publicar mi configuración
    sin revelar la ruta real dentro de mi sistema.
    
    Una vez instalados los paquetes necesarios dentro del contenedor y con
    el entorno que desee me puedo salir. Y ejecutar el siguiente comando
    para guardar los cambios:

    \texttt{podman commit --squash ubuntu-toolbox-22.04 localhost/my-ubuntu-toolbox:22.04}

    Una vez hecho esto, para crear el contenedor basta hacer
    \texttt{toolbox create --image my-ubuntu-toolbox:22.04}.
